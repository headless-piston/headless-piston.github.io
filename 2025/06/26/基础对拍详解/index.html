前言

对拍是 OI 中很有用且常用的工具。每当我们对着出题人用脚造的样例时，我们做题的心情都会受到很大影响。比如例 1，作为贪心题，样例过于特殊，导致可以放过各种神秘的假思路。例 2，可以很显然看出，出题人根本不想提供任何样例，换句话说，写对拍也是做这类题的必不可少的一部分。由此可见，对拍是一种非常重要的技能。限于笔者自身水平，本文仅介绍 Windows 系统下的对拍方法。Linux 系统下的一些操作欢迎各位神犇补充。

前置知识

随机数

对拍需要生成各种各样的数据。有时数据量极大，我们不可能一点点手动输入。所以，接下来介绍一些生成随机数的方法。
我们一般使用 rand() 函数来生成随机数（在头文件 <cstdlib> 中），这个函数有一个缺点是比较慢。在 Windows 系统下，它生成随机数的值域为 [0,215−1][0,2^{15}-1][0,215−1]，而在 Linux 系统下值域为 [0,231−1][0,2^{31}-1][0,231−1]。
值得注意的是，如果不初始化随机数种子，编译一次后每次运行程序得出的随机数是一样的，所以我们要使用 srand(seed) 函数来初始化随机数种子。一般选取当前系统时间为种子，方法是调用 <ctime> 里的 time() 函数。
解决 rand() 函数在 Windows 系统下值域过小的方式是使用 rand()<<15|rand()。

感谢 zzzyyyyhhhhh 指出不足之处。
time() 函数的返回值精确度是秒级别的。这导致我们快速多次运行程序生成的随机数种子是一样的。解决方法是直接将 time(0) 换为 __builtin_ia32_rdtsc()，这个函数的精确度达到了纳秒级别，具体取决于 CPU 的主频。

进阶

以上两种方法写出的程序稍显粗糙，我们可以加入更多功能，如时间限制和测试点编号。
我们使用 clock() 函数来获取从程序启动开始到调用该函数时占用的 CPU 时间。类型是 long，单位是毫秒。使用这个函数，我们可以方便地计算程序运行的用时，但要注意：这样做得到的只是一个近似值，不一定完全准确。
